# Agent OS Trial Operator Handoff Checklist

Source: [[Codex Threads as an Agent Operating System]]
Package: [[Agent OS Thread Orchestration Kit]]
Status: template-ready; evidence pending.

Use this before the next real Agent OS thread-map trial so the operator follows the evidence artifacts in the right order and does not upgrade public claims from memory.

## Operator sequence

1. Open [[Agent OS Thread Orchestration Kit/Agent OS Thread Map]] and choose one small real agent-system task.
2. Define the target workflow, thread roles, sensors, plugins, routing rules, and success criteria before the run starts.
3. During the run, capture proof links for inputs, tool calls, routing decisions, outputs, screenshots, logs, repo diffs, and failures.
4. Complete [[Agent OS Thread Orchestration Kit/Agent OS Promotion Decision Card]] using only captured proof.
5. Fill [[Agent OS Thread Orchestration Kit/Agent OS Post-Trial Debrief Template]] and list which README, prototype, infographic, or skill-draft claims are safe to patch.
6. Patch public claims only if the decision card and debrief both support the wording; otherwise keep the kit template-ready/evidence-pending.
