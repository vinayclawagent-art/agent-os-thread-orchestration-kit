# Agent OS Thread Orchestration Kit Prototype

Source: [[Codex Threads as an Agent Operating System]]

Latest improvement: 2026-06-20 added Trial Evidence Index so future thread-orchestration proof is mapped to specific README, prototype, and skill-draft claims before wording changes; evidence still pending.

## What it demonstrates
A lightweight architecture packet for designing agent systems as thread processes with sensors, plugins, routing, and evidence handoffs.

## How to use
1. Open [[Agent OS Thread Orchestration Kit/Trial Operator Handoff Checklist]] and assign the next real trial operator.
2. Open [[Agent OS Thread Orchestration Kit/Agent OS Thread Map]].
3. Fill it during one real task.
4. Attach concrete proof links before changing claims in the package README or skill draft.
5. Fill [[Agent OS Thread Orchestration Kit/Trial Evidence Index]] to map each proof item to the claim it supports.
6. Grade the proof with [[Agent OS Thread Orchestration Kit/Agent OS Evidence Quality Rubric]].
7. Complete [[Agent OS Thread Orchestration Kit/Agent OS Promotion Decision Card]] to decide promote / pilot-only / iterate / hold.
8. Fill [[Agent OS Thread Orchestration Kit/Agent OS Post-Trial Debrief Template]] before patching public claims.
